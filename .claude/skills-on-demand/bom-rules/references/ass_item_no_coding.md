## 定義

Ass Item No (組裝品料號) 是成品的唯一識別碼，包含產品型號、包裝方式、內部控制碼。

## 格式

```
{產品型號}_{包裝}{版本}_{5碼內部控制碼}
```

### 範例
- `1N4148_R1_00001` - 標準產品
- `1N4148-AU_R1_0Z0A1` - AU 等級產品

## 5碼內部控制碼解析

### 第1碼 - 有害物質
| 值 | 說明 |
|---|------|
| `0` | Halogen Free（無鹵）|
| `1` | Halogen contain（含鹵）|
| `V` | Lead-free glass（無鉛玻璃）|

### 第2碼 - 產品等級
| 值 | 說明 |
|---|------|
| `0` | Series Code / Standard AU |
| `Z` | AU+（車規加強版）|

### 第3碼 - 組裝廠/產品層級
| 值 | 說明 | ECR/ECN 納入 |
|---|------|:---:|
| `0` | 正規產品（新產品第一次發行）或廠內自製 | **是** |
| `1` | LE 產品（驗證僅 168H，僅開放特定客戶）| 否 |
| `2` | 外包產品（PJ 指定晶片委外代工）| 否 |
| `3` | 外購產品（委外生產）| 否 |
| `5` | PJ 自製 - MBU3 生產 | 否 |
| `6` | PJ 自製 - MBU2 生產 | 否 |
| `7` | PJ 自製 - MBU1 生產（從 `0` 拆分的新碼，同屬岡山廠）| **是** |
| `8` | ATEC 外包（使用者確認 2026-02-22）| 否 |

> **ECR/ECN 評估規則**：僅第3碼為 `0` 或 `7` 的料號納入變更評估。
> `0` 和 `7` 都是岡山廠自製（MBU1），`7` 是後來從 `0` 拆分出來的新碼。
> 其餘（LE/外包/外購/MBU2/MBU3/ATEC）均排除。
> 來源：使用者確認 2026-03-03。

### 第4-5碼 - 特殊需求

**Standard（第2碼=0）：**
| 值 | 說明 |
|---|------|
| `01` | Standard（新產品第一次發行）|
| `A1` | AU Standard BOM |
| `A2~A9` | Specific Requirement series code（特規續編）|
| `02, AX, Z1~Z7` | 已定義用途的保留碼 |

**AU+（第2碼=Z）：**
| 值 | 說明 |
|---|------|
| `A1` | AU+ Standard BOM |
| `A2~A9` | Specific Requirement series code（特規續編）|
| `02, AX, Z1~Z7` | 已定義用途的保留碼 |

**Customer Specific Requirement（CSR，Bx 起）：**
| 值 | 說明 |
|---|------|
| `B1~B9` | CSR（客戶待確認）|
| `C1` | CSR - Continental |
| `D1~E1` | CSR（客戶待確認）|
| `P1` | CSR - Aptiv |
| `V1` | CSR - Valeo |
| `Y1~Y9` | CSR（客戶待確認）|

## 使用場景

1. **產品識別**: 從料號解析產品規格和等級
2. **產線分配**: 根據第3碼判斷生產廠區
3. **客戶管理**: 識別客戶特規產品
4. **品質追溯**: 結合等級碼追蹤產品認證狀態
