# BOM 階層結構判斷規則（詳細）

> 規則 ID: BOM-HIERARCHY | 信心度: 中 | 更新: 2026-02-03

## 標準層級定義

- **Level 1**：成品（FG）— BOM 根節點，通常一筆，料號前綴 FG- 或產品代碼
- **Level 2**：子組件（SA）— 成品直接包含的模組/半成品，前綴 SA-
- **Level 3**：原物料（RM）— 子組件的直接材料，前綴 RM-
- **Level 4+**：子原物料 — 更細粒度，不是所有 BOM 都有

父子關係：每節點（除 Level 1）有 parent_part 指向上一層，一 parent 多 children。

## 例外 1：替代材料上提（信心度：中）

原物料缺貨時，替代件可能記錄在比預期更高的 BOM 層級。

**典型案例**：Level 3 的 RM-50100 缺貨，替代件 RM-50231 被放在 Level 2。

**判斷線索**：
1. 該層料號前綴與同層不一致（Level 2 出現 RM- 開頭）
2. 同 parent 下有兩筆以上功能相似材料
3. 備註含「替代」「OR」「SUB」關鍵字
4. 該料號在其他 BOM 版本出現在更深層級

**處理**：標記 `_is_substitute=1`、記錄 `_inferred_level`、若可確定原件記錄 `_substitute_for`。按推斷層級分析，不改原始資料。

## 例外 2：供應商壓平（信心度：低）

某些供應商的報價單/BOM 匯出會將深層材料壓平到較高層級（如 Level 3-4 全報在 Level 2）。

**判斷線索**：
1. 某 parent 下 children 數量異常多
2. children 料號前綴混雜多種類型
3. 與內部 BOM 結構明顯不同

**處理**：標記「疑似壓平」，需與內部 BOM 對照，不自動展開。

**待確認**：壓平時金額/數量是否已匯總？哪些供應商有此行為？

## 例外 3：BOM 版本差異（信心度：低）

同一成品可能有多個 BOM 版本（工程變更、設計修訂），結構可能不同。

**線索**：料號帶版本號（-01, -02, .A, .B）、同 parent 下新舊版材料並存。
