# /evolve — 系統自我演進審查

> 觸發 Architect 進行系統演進審查，識別改進機會。

## 指令說明

執行全面的系統架構審查，包括：
1. 分析最近的使用模式
2. 識別重複性任務
3. 檢查 Agent 覆蓋率
4. 建議架構改進
5. 提出新 Agent / 工具建議

## 使用方式

```
/evolve              # 標準審查
/evolve deep         # 深度審查（含所有歷史）
/evolve agent        # 專注於 Agent 優化
/evolve tools        # 專注於工具優化
```

## 執行流程

### 1. 收集數據
- 掃描 `shared/kb/evolution_log.md` 歷史
- 檢查 `shared/kb/dynamic/` 知識累積
- 分析當前會話的操作模式

### 2. 識別模式
- 重複操作（≥3 次相同類型）
- 失敗重試模式
- 手動操作瓶頸
- Agent 間頻繁切換

### 3. 產出報告
```markdown
## 演進審查報告 - {日期}

### 使用模式摘要
- 最常用 Agent: {列表}
- 常見任務類型: {列表}

### 識別的改進機會
1. {機會1}
2. {機會2}

### 建議行動
| 優先級 | 建議 | 預期效益 |
|-------|------|---------|
| 高 | {建議} | {效益} |

### 是否需要建立新 Agent？
{是/否 + 說明}
```

### 4. 使用者確認
- 呈報建議
- 等待使用者決定
- 執行已批准的改進

## 自動觸發條件

除了手動 `/evolve`，以下情況也會觸發審查建議：
- 同類型任務重複 5+ 次
- 新知識累積 10+ 條
- 新工具建立 3+ 個
- 使用者明確表示不滿意現有流程
